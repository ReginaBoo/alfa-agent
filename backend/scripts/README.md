# Тестовые данные для Jira

Скрипты для создания реалистичных тестовых данных в Jira для тестирования системы сбора метрик.

## 🚀 Быстрый старт

### 1. Настройка конфигурации

Откройте `config.py` и замените заглушки:

```python
# Ваш Jira instance (например: mycompany.atlassian.net)
JIRA_INSTANCE = "your-instance"

# Email администратора Jira
ADMIN_EMAIL = "admin@company.com"

# API Token администратора (получить: https://id.atlassian.com/manage-profile/security/api-tokens)
API_TOKEN = "your_api_token_here"

# Ваш email (будет руководителем всех проектов)
MY_EMAIL = "your.email@company.com"
```

### 2. Запуск скриптов

```bash
# 1. Создайте проекты (требуется права администратора)
python 02_create_test_projects.py

# 2. Создайте реалистичные тестовые данные
python 08_create_realistic_test_data.py

# 3. Синхронизируйте и проверьте метрики
python 04_sync_and_check.py
```

## 📁 Скрипты

| Файл | Описание |
|------|----------|
| `config.py` | Конфигурация (email, токены, проекты) |
| `02_create_test_projects.py` | Создание проектов (ты - руководитель) |
| `08_create_realistic_test_data.py` | Создание реалистичных задач с разными статусами |
| `04_sync_and_check.py` | Синхронизация и проверка метрик |

## 📊 Что создаётся

### Проекты (7 штук)

- **HEALTH** - здоровый проект (нормальная нагрузка)
- **CRUNCH** - перегруженный проект (много багов, быстрые сроки)
- **IMBAL** - несбалансированный (неравномерная загрузка)
- **IDLE** - недогруженный (долгие сроки)
- **BUGS** - много багов
- **KANBAN** - постоянное течение задач

### Задачи

- Разные типы: Task, Bug, Story
- Разные статусы: Created, In Progress, Review, Done
- Разные даты создания (от 1 до 120 дней назад)
- Разные временные интервалы между переходами
- Реалистичные описания для обучения модели
- Комментарии в changelog

## ⚠️ Требования

- Аккаунт администратора Jira
- API Token администратора
- Ваш аккаунт в Jira (MY_EMAIL)

## 🔧 Решение проблем

### Ошибка "Пользователь не может быть назначен для задач"

Убедитесь, что ваш аккаунт имеет права на создание и назначение задач в проектах.

### Задачи не создаются

Проверьте:
1. Правильность JIRA_INSTANCE
2. Валидность API_TOKEN
3. Права администратора у ADMIN_EMAIL

### Нет переходов статусов

Проверьте workflow проекта в Jira и убедитесь, что статусы соответствуют `PROJECT_WORKFLOWS` в `config.py`.
